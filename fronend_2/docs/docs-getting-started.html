Getting Started Libraries

We have used this libraries to develop a Adminpro Vuetify Admin Template. Before get started with this template let you know that which are the libraries we have used in this template.
VueJs

VueJS is a progressive framework for building user interfaces. Vuexy Admin template is build upon VueJS. It is mandatory to know what is VueJS, If you don't know what is VueJS and how to use it, we highly recommend learninng VueJS before proceeding.

VueCLI

Vue CLI is a full system for rapid Vue.js development. Vue CLI aims to be the standard tooling baseline for the Vue ecosystem.

Vuetify

Vuetify is a library of Vuejs components that facilitates front-end development. It is beautifully designed and pleasant looking framework.

Getting Started

Note: we recomonded you to please make your one own css file & and add that in your page, so whenever the update of Adminpro Vuetify admin comes it does not affect your code.
Go to the downloaded package
Install Node.js From https://nodejs.org/en/download/ [Please install recommended version]
After that open command promt or any other terminal and go to Package Path.

cd [root] / package /

Npm is a default package manager for the JavaScript runtime environment Node.js. If you've already then update once.

npm install --global npm@latest

To check weather is node succesfully install or not.

npm --version
For yarn package manager.

npm install --global yarn

To check weather is yarn succesfully install or not.

yarn --version
Install VueCLI Globally.

npm install -g @vue/cli

For Yarn command is here

yarn global add @vue/cli
once you download the package, please extract the zip and you will get this folders:
documentation
landingpage
packages
Now, Go to packages folder and choose main or starterkit demo and do npm install.
To run the templates locally run below command in to your terminal. Open http://localhost:8080 to view it in the browser. The page will automatically reload if you make changes to the code.

npm run serve

Note: Sometimes app may run on different port than 8080. To be sure, You can check correct address in console right after label App running at: .
