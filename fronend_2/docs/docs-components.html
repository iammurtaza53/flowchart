Components

1. Alert

The v-alert component is used to convey important information to the user through the use contextual types icons and color. These default types come in in 4 variations: success, info, warning, and error. Default icons are assigned which help represent different actions each type portrays. Many parts of an alert such as border, icon, and color can also be customized to fit almost any situation.
Link Here:https://vuetifyjs.com/en/components/alerts/

2. Application

In Vuetify, the v-app component and the app prop on components like v-navigation-drawer, v-app-bar, v-footer and more, help bootstrap your application with the proper sizing around component. This allows you to create truly unique interfaces without the hassle of managing your layout sizing. The v-app component is REQUIRED for all applications. This is the mount point for many of Vuetify's components and functionality and ensures that it propagates the default application variant (dark/light) to children components and also ensures proper cross-browser support for certain click events in browsers like Safari. v-app should only be used within your application ONCE.
Link Here:https://vuetifyjs.com/en/components/application/

3. Aspect Ratios

The v-responsive component can be used to fix any section to a specific aspect ratio
Link Here:https://vuetifyjs.com/en/components/aspect-ratios/

4. Avatars

The v-avatar component is typically used to display circular user profile pictures. This component will allow you to dynamically size and add a border radius of responsive images, icons, and text. A tile variation is available for displaying an avatar without border radius.
Link Here:https://vuetifyjs.com/en/components/avatars/

5. Badges

The v-badge component superscripts or subscripts an avatar-like icon or text onto content to highlight information to a user or to just draw attention to a specific element. Content within the badge usually contains numbers or icons.
Link Here:https://vuetifyjs.com/en/components/badges/

6. Banners

The v-banner component is used as middle-interruptive message to user with 1-2 actions. It comes in 2 variations, single-line and multi-line (implicit). These can have icons which you can use with your message and actions.
Link Here:https://vuetifyjs.com/en/components/banners/

7. App Bars

The v-app-bar component is pivotal to any graphical user interface (GUI), as it generally is the primary source of site navigation. The app-bar component works great in conjunction with a v-navigation-drawer for providing site navigation in your application.
Link Here:https://vuetifyjs.com/en/components/app-bars/

8. Toolbars

The v-toolbar component is pivotal to any gui, as it generally is the primary source of site navigation. The toolbar component works great in conjunction with v-navigation-drawer and v-card.
Link Here:https://vuetifyjs.com/en/components/toolbars/

9. System bars

The v-system-bar component can be used for displaying statuses to the user. It looks like the Android system bar and can contain icons, spacers, and some text.
Link Here:https://vuetifyjs.com/en/components/system-bars/

10. Bottom navigation

The v-bottom-navigation is an alternative to the sidebar. It is primarily used on mobile and comes in two variants, icons and text, and shift.
Link Here:https://vuetifyjs.com/en/components/bottom-navigation/

11. Bottom sheets

The bottom sheet is a modified v-dialog that slides from the bottom of the screen, similar to a v-bottom-navigation. Whereas a bottom navigation component is for buttons and specific application level actions, a bottom sheet can contain anything.
Link Here:https://vuetifyjs.com/en/components/bottom-sheets/

12. Breadcrumbs

The v-breadcrumbs component is a navigational helper for pages. It can accept a Material Icons icon or text characters as a divider. An array of objects can be passed to the items property of the component. Additionally, slots exists for more control of the breadcrumbs, either utilizing v-breadcrumbs-item or other custom markup.
Link Here:https://vuetifyjs.com/en/components/breadcrumbs/

13. Buttons

The v-btn component replaces the standard html button with a material design theme and a multitude of options. Any color helper class can be used to alter the background or text color.
Link Here:https://vuetifyjs.com/en/components/buttons/

14. Buttons: Floating Action Button

The v-btn component can be used as a floating action button. This provides an application with a main point of action. Combined with the v-speed-dial component, you can create a diverse set of functions available for your users.
Link Here:https://vuetifyjs.com/en/components/floating-action-buttons/

15. Calendars

The v-calendar component is used to display information in a daily, weekly, or monthly view. The daily view has slots for all day or timed elements, and the weekly and monthly view has a slot for each day. Optionally you can pass in an array of events and they will be rendered over the appropriate days and times.
Link Here:https://vuetifyjs.com/en/components/calendars/

16. Cards

The v-card component is a versatile component that can be used for anything from a panel to a static image. The card component has numerous helper components to make markup as easy as possible. Components that have no listed options use Vue's functional component option for faster rendering and serve as markup sugar to make building easier.
Link Here:https://vuetifyjs.com/en/components/cards/

17. Carousels

The v-carousel component is used to display large numbers of visual content on a rotating timer.
Link Here:https://vuetifyjs.com/en/components/carousels/

18. Chips

The v-chip component is used to convey small pieces of information. Using the close property, the chip becomes interactive, allowing user interaction. This component is used by the v-chip-group for advanced selection options.
Link Here:https://vuetifyjs.com/en/components/chips/

19. Dialogs

The v-dialog component inform users about a specific task and may contain critical information, require decisions, or involve multiple tasks. Use dialogs sparingly because they are interruptive.
Link Here:https://vuetifyjs.com/en/components/dialogs/

20. Dividers

The v-divider component is used to separate sections of lists or layouts.
Link Here:https://vuetifyjs.com/en/components/dividers/

21. Expansion panels

The v-expansion-panel component is useful for reducing vertical space with large amounts of information. The default functionality of the component is to only display one expansion-panel body at a time; however, with the multiple property, the expansion-panel can remain open until explicitly closed.
Link Here:https://vuetifyjs.com/en/components/expansion-panels/

22. Footers

The v-footer component is used for displaying general information that a user might want to access from any page within your site.
Link Here:https://vuetifyjs.com/en/components/footer/

23. Button groups

The v-btn-toggle component is a simple wrapper for v-item-group built specifically to work with v-btn.
Link Here:https://vuetifyjs.com/en/components/button-groups/

24. Chip groups

The v-chip-group supercharges the v-chip component by providing groupable functionality. It is used for creating groups of selections using chips.
Link Here:https://vuetifyjs.com/en/components/chip-groups/

25. Item groups

The v-item-group provides the ability to create a group of selectable items out of any component. This is the baseline functionality for components such as v-tabs and v-carousel.
Link Here:https://vuetifyjs.com/en/components/item-groups/

25. List item groups

The v-list-item-group provides the ability to create a group of selectable v-list-items. The v-list-item-group component utilizes v-item-group at its core to provide a clean interface for interactive lists
Link Here:https://vuetifyjs.com/en/components/list-item-groups/

26. Slide groups

The v-slide-group component is used to display pseudo paginated information. It uses v-item-group at its core and provides a baseline for components such as v-tabs and v-chip-group.
Link Here:https://vuetifyjs.com/en/components/slide-groups/

27. Windows

The v-window component provides the baseline functionality for transitioning content from 1 pane to another. Other components such as v-tabs, v-carousel and v-stepper utilize this component at their core.
Link Here:https://vuetifyjs.com/en/components/windows/

28. Hover

The v-hover component provides a clean interface for handling hover states for any component.
Link Here:https://vuetifyjs.com/en/components/hover/

29. Icons

The v-icon component provides a large set of glyphs to provide context to various aspects of your application. For a list of all available icons, visit the official Material Design Icons page. To use any of these icons simply use the mdi- prefix followed by the icon name.
Link Here:https://vuetifyjs.com/en/components/icons/

30. Images

The v-img component is packed with features to support rich media. Combined with the vuetify-loader, you can add dynamic progressive images to provide a better user experience.
Link Here:https://vuetifyjs.com/en/components/images/

31. Lazy loading

The v-lazy component is used to dynamically load components based upon an elements visiblity.
Link Here:https://vuetifyjs.com/en/components/lazy/

32. Lists

The v-list component is used to display information. It can contain an avatar, content, actions, subheaders and much more. Lists present content in a way that makes it easy to identify a specific item in a collection. They provide a consistent styling for organizing groups of text and images.
Link Here:https://vuetifyjs.com/en/components/lists/

33. Menus

The v-menu component shows a menu at the position of the element used to activate it.
Link Here:https://vuetifyjs.com/en/components/menus/

34. Navigation drawers

The v-navigation-drawer component is what your users will utilize to navigate through the application. The navigation-drawer is pre-configured to work with or without vue-router right out the box. For the purpose of display, some examples are wrapped in a v-card element. Within your application you will generally place the v-navigation-drawer as a direct child of v-app.
Link Here:https://vuetifyjs.com/en/components/navigation-drawers/

35. Overlays

The v-overlay component is used to provide emphasis on a particular element or parts of it. It signals to the user of a state change within the application and can be used for creating loaders, dialogs and more.
Link Here:https://vuetifyjs.com/en/components/overlays/

36. Paginations

The v-pagination component is used to separate long sets of data so that it is easier for a user to consume information. Depending on the length provided, the pagination component will automatically scale. To maintain the current page, simply supply a v-model attribute.
Link Here:https://vuetifyjs.com/en/components/paginations/

37. Parallax

The v-parallax component creates a 3d effect that makes an image appear to scroll slower than the window.
Link Here:https://vuetifyjs.com/en/components/parallax/

38. Color pickers

The v-color-picker allows you to select a color using a variety of input methods.
Link Here:https://vuetifyjs.com/en/components/color-pickers/

39. Date/month pickers

The v-date-picker is stand-alone component that can be utilized in many existing Vuetify components. It offers the user a visual representation for selecting date/month.
Link Here:https://vuetifyjs.com/en/components/date-pickers/

40. Time pickers

The v-time-picker is stand-alone component that can be utilized in many existing Vuetify components. It offers the user a visual representation for selecting the time.
Link Here:https://vuetifyjs.com/en/components/time-pickers/

41. Progress circular

The v-progress-circular component is used to convey data circularly to users. It also can be put into an indeterminate state to portray loading.
Link Here:https://vuetifyjs.com/en/components/progress-circular/

42. Progress linear

The v-progress-linear component is used to convey data visually to users. They can also represent an indeterminate amount, such as loading or processing.
Link Here:https://vuetifyjs.com/en/components/progress-linear/

43.Ratings

The rating component is a specialized but crucial piece in building user widgets. Collecting user feedback via ratings is a simple analytic that can provide a lot of feedback to your product or application.
Link Here:https://vuetifyjs.com/en/components/ratings/

44. Sheets

The v-sheet is designed to power other paper components within Vuetify. It is meant to be used as a low level component.
Link Here:https://vuetifyjs.com/en/components/sheets/

45. Skeleton loaders

The v-skeleton-loader component is a versatile tool that can fill many roles within a project. At its heart, the component provides an indication to the user that something is coming but not yet available. There are over 30 pre-defined options available that can be combined to make custom examples.
Link Here:https://vuetifyjs.com/en/components/skeleton-loaders/

46. Snackbars

The v-snackbar component is used to display a quick message to a user. Snackbars support positioning, removal delay, and callbacks.
Link Here:https://vuetifyjs.com/en/components/snackbars/

47. Sparklines

The sparkline component can be used to create simple graphs, like GitHub's contribution chart. Any SVG attribute may be used in addition to the ones listed below.
Link Here:https://vuetifyjs.com/en/components/sparklines/

48. Steppers

The v-stepper component displays progress through numbered steps.
Link Here:https://vuetifyjs.com/en/components/steppers/

49. Subheaders

The v-subheader component is used to separate sections of lists.
Link Here:https://vuetifyjs.com/en/components/subheaders/

50. Tabs

The v-tabs component is used for hiding content behind a selectable item. This can also be used as a pseudo-navigation for a page, where the tabs are links and the tab-items are the content.
Link Here:https://vuetifyjs.com/en/components/tabs/

51. Timelines

The v-timeline is useful for stylistically displaying chronological information.
Link Here:https://vuetifyjs.com/en/components/timelines/

52. Tooltips

The v-tooltip component is useful for conveying information when a user hovers over an element. You can also programmatically control the display of tooltips through a v-model. When activated, tooltips display a text label identifying an element, such as a description of its function.
Link Here:https://vuetifyjs.com/en/components/tooltips/

53. Treeview

The v-treeview component is useful for displaying large amounts of nested data.
Link Here:https://vuetifyjs.com/en/components/treeview/
